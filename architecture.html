数据流

Database Change
  → Room 自动通知 Flow
  → DAO 发射新数据
  → Repository 映射为 Domain Model (Entity → Domain)
  → ViewModel StateFlow 更新
  → Compose collectAsState() 感知变化
  → UI 自动重组

整个过程全自动响应式，无需手动刷新。数据变化时 UI 自动更新。

状态管理模式

@HiltViewModel
class BillViewModel @Inject constructor(
    private val repository: TransactionRepository
) : ViewModel() {
    // 内部可变状态
    private val _uiState = MutableStateFlow(BillUiState())
    // 对外只读
    val uiState: StateFlow<BillUiState> = _uiState.asStateFlow()

    // 响应式数据管道：UI状态变化 → 自动重新查询
    val dayGroups: StateFlow<List<DayGroup>> = _uiState
        .flatMapLatest { state -> repository.getFiltered(...) }
        .stateIn(viewModelScope, WhileSubscribed(5000), emptyList())
}

关键设计决策

日期归一化

所有交易日期存储前归一化到当天零点。因为 SQLite GROUP BY date 按时间戳分组，同时刻不同时间的记录会被分到不同组。归一化确保同一天正确聚合。

Flow 响应式查询

DAO 返回 Flow<List<T>> 而非一次性 List。数据库变化时自动通知 ViewModel → UI 重组。整个链路无需手动刷新。

Entity → Domain 映射

Repository 负责 Entity ↔ Domain Model 转换。Domain 层不依赖 Room 注解，保持纯 Kotlin。

原子状态更新

使用 _uiState.update { it.copy(...) } 而非 _uiState.value =。原子操作防止并发消息丢失——24 处全量修改。

工程质量保障

🧵

线程安全

  • · SimpleDateFormat 每次创建新实例（非线程安全类不共享）
  • · Room DAO suspend 函数自动后台线程
  • · Bitmap 解码使用 Dispatchers.IO + 缩放后回收原图
  • · 网络请求 withContext(Dispatchers.IO) 包裹
📅

日期归一化

  • · 所有 date 存储时归一化到当天 00:00:00.000
  • · SQLite GROUP BY date 同天正确聚合
  • · 导出时用 createdAt（精确时间戳）保证时间精度
🔄

Entity ↔ Domain 映射

  • · Repository 层负责 Entity → Domain Model 转换
  • · Domain 层不依赖 Room 注解，纯 Kotlin
  • · LEFT JOIN 查询用 TransactionWithCategory 承载分类信息
🛡️

安全与混淆

  • · Release 启用 ProGuard 混淆 + 资源压缩
  • · DTO/Retrofit/序列化完整 keep 规则
  • · API Key 不硬编码，用户自行配置
  • · HTTP 日志仅 DEBUG 模式输出
