1. AI 自然语言记账系统

Prompt 工程 · 多条目解析 · 5级分类匹配 · 对话纠错

构建了 140 行 System Prompt，包含完整的上下文工程：

  • 分类列表动态注入（所有可用分类名）
  • 日期参考上下文（今天/昨天/前天/本周一的精确日期）
  • 8 条日期推断规则（"刚刚/中午/昨晚/上周X/X天前"等）
  • 6 条金额推断规则（"两千/十几块/一百来块"等口语化金额）
  • 12 条分类推断规则（"吃饭外卖奶茶→餐饮，滴滴地铁→交通"等关键词映射）
  • 10 个多样化示例覆盖单笔/多笔/收入/转账/口语化场景
用户："午饭20，打车10，给妈妈转500红包" → AI 解析为 3 条结构化记录 → 自动分类：餐饮/交通/人情 → 用户确认 → 3 条同时入库

5 级分类匹配策略

1. 精确匹配: AI返回"餐饮" = 本地"餐饮" ✓
2. 包含匹配: 分类名包含输入（如本地"餐饮消费"包含AI返回的"餐饮"）
3. 被包含匹配: 输入包含分类名（AI返回"交通费用"，本地有"交通"）
4. 归一化匹配: 去除"费/花/销/开/支/用"后缀后比较，如"交通费"→"交通"
5. 兜底: 匹配到"其他"分类

对话式纠错

检测 9 种纠错关键词（不对/错了/改成/应该是/不是/换成/修改/分类/金额），自动触发纠错模式：AI 保留之前解析结果的未提及字段，只修改用户指定的字段。纠错 Prompt 包含上一次的完整解析结果作为上下文。

DeepSeek API 双模型 Fallback

主模型: deepseek-v4-flash (temperature=0.1, 快速记账) 备用模型: deepseek-chat (temperature=0.1, v4-flash失败时自动切换)

2. 多模态输入统一管线

文字 · 拍照OCR · 语音 → 统一 AI 解析

📝 文字 → ParseTransactionUseCase.invoke(text, categories) 📷 拍照 → ML Kit OCR → 提取文字 → ParseTransactionUseCase.parseFromImage(bitmap, categories) 🎤 语音 → SpeechRecognizer → 文字 → ParseTransactionUseCase.invoke(spokenText, categories)

ML Kit OCR：使用 ChineseTextRecognizerOptions 中文模型，图片压缩至 1024px，后台线程解码。

语音识别：SpeechRecognizer 直接 API（非系统对话框），自定义语音 UI（扩散光环 + 20条声波柱），友好错误提示。

3. 自绘 Canvas 图表系统

零第三方依赖 · Catmull-Rom 平滑曲线 · 触摸交互

折线图（多数据点）：

// Catmull-Rom 平滑曲线，控制点 Y 钳制避免过度摆动 val cp1y = (p1.y + (p2.y - p0.y) / 6f).coerceIn(minY, maxY) val cp2y = (p2.y - (p3.y - p1.y) / 6f).coerceIn(minY, maxY) path.cubicTo(p1.x + (p2.x - p0.x) / 6f, cp1y, p2.x - (p3.x - p1.x) / 6f, cp2y, p2.x, p2.y) // 渐变填充 + 虚线网格 + 数据点圆点 + 点击 Tooltip

柱状图（单数据点—日视图）：双柱对比（支出珊瑚渐变 + 收入绿色渐变），柱顶数字标签，点击弹性缩放 1.08x

X 轴自适应：entries ≤ 12 全显，>12 ≤ 20 隔一显，>20 隔二显。网格线和文字使用 Material 主题色值

交互：`detectTapGestures` 检测点击 → 高亮数据点 → 显示详情 Panel（日期+支出+收入）

4 表 · 外键 · 复合索引 · Flow 响应式查询

表名说明关键字段索引
transactions交易记录amount, type, category_id(FK), note, datedate, type, category_id, (date,type)
categories收支分类name, icon_name, color, is_income, is_default—

attachments图片附件transaction_id(FK), file_pathtransaction_id

核心查询

  • 按日聚合：strftime('%Y-%m-%d', date/1000, 'unixepoch', 'localtime') 按本地时区分组
  • 按分类聚合：LEFT JOIN categories + GROUP BY category_id 用于饼图和排行
  • 多条件筛选：7 条件可选组合（分类/类型/金额区间/关键词/日期），利用 IS NULL OR 模式避免拼接 SQL
  • 日期归一化：所有 date 归一化到当天零点，确保 GROUP BY 同天正确聚合

预置 16 个分类

首次创建数据库时通过 RoomDatabase.Callback().onCreate() 自动插入：11 个支出分类（餐饮/交通/购物/娱乐/居住/通讯/医疗/教育/人情/宠物/其他）+ 5 个收入分类（工资/兼职/理财/红包/其他收入）
